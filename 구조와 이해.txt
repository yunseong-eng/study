1. Java
Java 는 범용 프로그래밍 언어입니다. 여기서 '범용'이란, 자바가 여러 분야에서 다양하게 사용할 수 있다는 뜻이에요. 즉, 게임, 웹사이트, 앱 등 많은 프로그램을 만들 수 있어요. 자바를 사용해서 여러가지 프로그램을 만들 수 있는데, 이클립스에서 자바로 코드를 짜면 주로 콘솔 창에 출력 결과값이 나옵니다. 자바는 주로 서버 측 로직, 즉 백엔드 개발에 많이 사용됩니다.(서버 측 로직은, 서버라는 컴퓨터가 사용자들의 요청을 받아서 처리하는 역할이에요. 예를 들어, 우리가 웹사이트에 접속하면, 서버가 요청을 받아서 우리가 원하는 정보를 보여주거나 처리해주는 거죠. 자바는 이런 서버의 역할을 하는 프로그램을 만드는데 많이 사용돼요. 이걸 백엔드 개발이라고 해요. 백엔드는 눈에 보이지 않지만, 서버에서 일어나는 모든 중요한 작업을 담당해요. 즉, 서버가 정보를 저장하고, 처리하고, 사용자에게 필요한 데이터를 보내는 일을 자바가 해요.)
이 말은 자바가 데이터 처리, 비즈니스로직, 데이터베이스와의 상호작용 등을 담당한다는 것입니다.

2.데이터베이스(DB) 와 SQL
데이터베이스(DB) 는 데이터를 저장하고 관리하는 공간입니다. 예를들어 회원정보나 상품정보 등을 저장할 수 있습니다. 자바 프로그램에서 DB와 연동하면, 자바코드로 DB에 데이터를 추가, 수정,삭제,조회 할 수 있습니다.
- SQL은 데이터베이스에서 데이터를 조회하거나 수정할 때 사용하는 언어입니다. 

3. HTML (HyperText Markup Language)
HTML은 웹 페이지의 뼈대를 구성하는 언어입니다. 웹 페이지의 구조를 정의하며, 제목, 단락, 링크, 이미지 등 페이지에 표시될 요소들을 지정합니다. 쉽게말해, HTML은 웹사이트의 "뼈대"를 만드는 역할을 합니다.
예를들어 <h1> <p> <a> 같은 태그를 통해 제목, 문단, 링크 등을 표시합니다.

4. CSS (Cascading Style Sheets)
CSS는 HTML로 만든 웹 페이지의 디자인을 담당합니다.
즉 CSS는 웹페이지를 이쁘게 꾸미는 역할을 합니다.

5.JavaScript
JavaScript는 웹 페이지에 동적인 기능을 추가하는 스크립트 언어입니다. 예를들어 사용자가 버튼을 클릭했을 때 무언가가 일어나게 하거나 입력한 데이터를 검증하는 역할을 합니다. (여기서 "입력한 데이터를 검증한다"는 것은, 사용자가 입력한 정보가 올바른지 확인하는 걸 말해요. 예를 들어, 회원 가입할 때 비밀번호를 입력하는데, 자바스크립트가 그 비밀번호가 충분히 안전한지, 혹은 이메일 주소가 올바른 형식인지 미리 확인해주는 거예요.)
자바스크립트는 웹 브라우저에서 실행되며, 웹 페이지의 상호작용을 관리합니다. ("상호작용을 관리한다"는 것은 사용자가 웹 페이지에서 무언가를 클릭하거나 스크롤할 때, 그에 맞춰 웹 페이지가 반응하도록 만드는 걸 말해요. 예를 들어, 버튼을 누르면 창이 열리거나, 메뉴가 내려오는 것처럼 사용자의 행동에 따라 웹 페이지가 반응하게 만드는 거예요.)

6. jQuery
jQuery는 자바스크립트를 더 쉽게 쓸 수 있게 해주는 라이브러리입니다. (여기서 '라이브러리'란, 자주 쓰이는 코드들을 모아둔 도구상자 같은 거예요. 이 도구상자를 쓰면 복잡한 자바스크립트 코드를 간단하게 쓸 수 있어요) 예를들어 자바 스크립트로 해야 할 일을 더 간단하게 쓸 수 있도록 도와줍니다. 
예를들어 클릭이벤트나 애니메이션을 쉽게 처리할 수 있도록 도와줍니다.

7.AJAX (Asynchronous JavaScript and XML)
AJAX는 페이지를 새로고침 하지 않고도 서버와 데이터를 주고받을 수 있는 기술입니다.
예를들어 게시글을 작성하고 페이지를 새로고침 하지 않아도 글이 게시되도록 하는 것입니다.
jQuery로 AJAX를 쉽게 사용할 수 있습니다.

정리하자면:
Java: 서버 로직과 DB와의 연동, 백엔드 작업.
DB: 데이터를 저장하는 공간, SQL로 조작.
HTML: 웹 페이지의 구조(뼈대) 정의.
CSS: 웹 페이지의 디자인과 레이아웃 설정.
JavaScript: 웹 페이지의 동적 기능, 상호작용 추가.
jQuery: 자바스크립트를 더 쉽게 쓰게 도와주는 도구.
AJAX: 새로고침 없이 서버와 데이터 통신 가능.

쉽게 비유를 하자면
웹 사이트는 건물이라고 하고 HTML은 건물의 구조나 골격을, CSS는 그 건물의 꾸미는 외관을,
JavaScript는 자동문이나 전등 같은 동작하는 기능을, DB는 건물 안의 서류나 물건을 저장하는 창고를 나타냅니다. 그리고 Java는 건물 안에서 작동하는 모든 시스템(서버 역할)을 담당한다고 생각하면 됩니다.
